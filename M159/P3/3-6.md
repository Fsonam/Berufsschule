# Anleitung: Gruppenrichtlinien (GPO) – Ordnerumleitung, Netzwerk-Wartezeit, Sicherheitsrichtlinien

## Zielsetzung

Einrichten funktionierender Gruppenrichtlinien für:

* Ordnerumleitung „Documents“
* Domänenweite Richtlinie „Warten auf Netzwerk“
* Registry-Sperre
* Bildschirmschoner-Einstellungen mit Passwortschutz

---

## Vorbereitung

* OU-Struktur vorhanden
* Client und DC verbunden
* Group Policy Management installiert

---

## Ordnerumleitung „Documents“

### GPO erstellen

Name:
Ordner 'Documents' wird umgeleitet

### Konfiguration

User Configuration → Policies → Windows Settings → Folder Redirection → Documents

Einstellung:
* Redirect to: Home Folder

### Verknüpfung

Intern → Houston → Informatik → Users

### Test

* Benutzer anmelden
* „Documents“ öffnet Home-Ordner auf Server

---

## GPO: „Immer auf Netzwerk warten“

### GPO erstellen

Name:
Bei Neustart und Anmeldung immer auf Netzwerk warten

### Konfiguration

Computer Configuration → Administrative Templates → System → Logon

Einstellung:
* Always wait for the network → Enabled

### Verknüpfung

* Domänen-Root (work.wondertoys.local)

---

## GPO: Registry-Sperre & Bildschirmschoner

### GPO erstellen

Name:
Kein Registryzugriff & Kennwortschutz

### Bildschirmschoner

User Configuration → Administrative Templates → Control Panel → Personalization

* Screen saver timeout: 60 Sekunden
* Password protect the screen saver: Enabled

### Registry sperren

User Configuration → Administrative Templates → System

* Prevent access to registry editing tools: Enabled

### Verknüpfung

Intern → Houston → Informatik → Users

---

## Tests

* Registry kann nicht mehr geöffnet werden
* Bildschirmschoner startet nach 60 Sekunden
* Passwort erforderlich bei Reaktivierung
* gpupdate /force funktioniert

---

## Kontrolle

* Alle GPOs greifen korrekt
* Keine Konflikte
* Benutzer-Einstellungen werden angewendet
