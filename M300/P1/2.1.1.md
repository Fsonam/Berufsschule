# Aufgabe 2.1.1 – Domänencontroller mit Samba (Vorbereitung)
## Ziel
Vorbereitung einer Linux-VM für den Einsatz als Samba-Domänencontroller.

## Komplette Schritt-für-Schritt Anleitung
### 1. VM starten
Linux-VM starten und einloggen:
```bash
Benutzer: vmadmin
Passwort: Riethuesli>12345
```
### 2. Firewall starten & Internet testen
Firewall-VM starten und testen:
```bash
ping www.gbssg.ch
```
Wenn du eine Antwort bekommst → alles OK  
Wenn nicht → Netzwerk prüfen
### 3. Domäne festlegen
Format:
```text
<deinname>.m300
```
Beispiel:
```text
mueller.m300
```
### 4. Hostname setzen
```bash
sudo hostnamectl set-hostname dc1
```
### 5. Hosts-Datei konfigurieren
```bash
sudo nano /etc/hosts
```
Eintrag hinzufügen:
```text
192.168.210.41 dc1.<deinname>.m300 dc1
```
### 6. Netzwerk konfigurieren (Netplan)
```bash
sudo nano /etc/netplan/01-netcfg.yaml
```
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    ens33:
      addresses:
        - 192.168.210.41/24
      routes:
        - to: default
          via: 192.168.210.1
      nameservers:
        addresses: [192.168.210.1]
```
### 7. Netzwerk anwenden
```bash
sudo netplan apply
```
### 8. Netzwerk testen
```bash
ip a
ping 8.8.8.8
ping google.com
```
### 9. System aktualisieren
```bash
sudo apt update
sudo apt upgrade -y
```
### 10. Neustart
```bash
reboot
```
